Catalogo API


L'applicazione dispone di servizi API in stile REST che permettono ad altre applicazioni o script di accedere ai valori di particolato registrati dalle centraline Luftdaten. Il servizio permette l'accesso ai valori di particolato, sia realtime sia storici, per una località desiderata, grazie ad un paradigma di URL statico illustrato di seguito. Eventuali errori nell'utilizzo delle API vengono segnalati attraverso codici di risposta HTTP. Per tutte le richieste e le risposte dall'API viene utilizzato il formato JSON.

Di seguito è mostrato il paradigma delle API e delle relative informazioni associate.


Liste di elementi:


Queste API mostrano la lista dei dati associati a tutte le località contenute nel database.


Lista di tutte le località
{{ request.get_host }}{% url 'cities_list' %}


Lista dei valori real-time del particolato per tutte le località
{{ request.get_host }}{% url 'realtime_data' %}


Lista dei valori storici del particolato per tutte le località
{{ request.get_host }}{% url 'historical_data' %}


Elementi al dettaglio:


Queste API mostrano la lista dei dati associati a una specifica località contenuta nel database. Ogni località è identificata da un corrispondente ID.

È possibile reperire dati associati ad una specifica località contenuta nel database inserendo il suo ID nell'endpoint dell'API, avendo cura di inserirlo nella posizione corretta. Il catalogo delle località con relativo ID è consultabile a questo link.


Parametri di definizione di una specifica località
{{ request.get_host }}{% url 'city_detail' pk=1 %}


Valori real-time del particolato di una specifica località
{{ request.get_host }}{% url 'realtime_data_detail' pk=1 %}


Serie storica del particolato di una specifica località
{{ request.get_host }}{% url 'time_serie_detail' pk=1 %}
